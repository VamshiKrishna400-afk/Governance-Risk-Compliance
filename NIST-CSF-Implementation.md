# NIST Cybersecurity Framework (CSF) 2.0 – Basic Implementation Guide

This guide provides a practical implementation checklist based on the **NIST Cybersecurity Framework (CSF) 2.0**. It is organized according to the six core CSF functions.

---

# 1. Govern

Establish cybersecurity governance and define how security will be managed across the organization.

## Objectives

- Define cybersecurity policies and procedures.
- Assign clear roles and responsibilities for security.
- Identify accountability for cybersecurity decisions.
- Assess business and cybersecurity risks regularly.
- Ensure compliance with applicable laws, regulations, and industry standards.
- Establish an incident reporting and escalation process.
- Review and update cybersecurity policies periodically.
- Promote a culture of cybersecurity awareness throughout the organization.

### Implementation Checklist

- [ ] Create cybersecurity policies.
- [ ] Define security roles and responsibilities.
- [ ] Assign executive accountability.
- [ ] Perform periodic risk assessments.
- [ ] Maintain regulatory compliance.
- [ ] Document incident escalation procedures.
- [ ] Review policies annually or after major business changes.
- [ ] Conduct organization-wide security awareness initiatives.

---

# 2. Identify

Understand what needs to be protected by identifying organizational assets, data, and risks.

## Objectives

- Create and maintain asset inventories.
- Identify critical business systems.
- Classify sensitive information.
- Understand data flows.
- Assess third-party cybersecurity risks.
- Perform regular risk assessments.
- Document critical business processes.

### Implementation Checklist

- [ ] Maintain hardware inventory.
- [ ] Maintain software inventory.
- [ ] Inventory cloud services.
- [ ] Identify critical systems.
- [ ] Classify sensitive information.
- [ ] Document data flows.
- [ ] Assess third-party vendors.
- [ ] Perform vulnerability and risk assessments.
- [ ] Document business processes and supporting technology.

---

# 3. Protect

Implement safeguards that reduce the likelihood and impact of cybersecurity incidents.

## Objectives

- Apply least privilege.
- Keep systems updated.
- Protect endpoints.
- Enable built-in security features.
- Secure backups.
- Implement Multi-Factor Authentication (MFA).
- Train employees.
- Manage user lifecycle securely.

### Implementation Checklist

- [ ] Implement Role-Based Access Control (RBAC).
- [ ] Enforce Principle of Least Privilege.
- [ ] Apply operating system updates.
- [ ] Patch applications regularly.
- [ ] Update firmware.
- [ ] Deploy antivirus or endpoint protection.
- [ ] Enable host firewalls.
- [ ] Enable device encryption.
- [ ] Enable Secure Boot where supported.
- [ ] Perform regular backups.
- [ ] Store backups securely.
- [ ] Enable MFA for administrators.
- [ ] Enable MFA for remote access.
- [ ] Conduct phishing awareness training.
- [ ] Conduct social engineering awareness training.
- [ ] Establish onboarding procedures.
- [ ] Establish offboarding procedures.

---

# 4. Detect

Continuously monitor systems to detect cybersecurity events as early as possible.

## Objectives

- Detect suspicious activity.
- Monitor infrastructure.
- Collect security logs.
- Monitor endpoints.
- Report suspicious activity.

### Implementation Checklist

- [ ] Configure security alerts.
- [ ] Deploy Intrusion Detection Systems (IDS).
- [ ] Deploy Intrusion Prevention Systems (IPS).
- [ ] Implement SIEM.
- [ ] Deploy Endpoint Detection and Response (EDR).
- [ ] Monitor user activity.
- [ ] Monitor network activity.
- [ ] Monitor system logs.
- [ ] Establish phishing reporting procedures.
- [ ] Ensure employees know how to contact IT/security.

---

# 5. Respond

Develop and maintain an incident response capability that minimizes business impact.

## Objectives

- Build an Incident Response Plan.
- Define response responsibilities.
- Establish communication procedures.
- Practice incident response.
- Improve continuously.

## Incident Response Plan

Clearly define:

- Response actions
- Response leadership
- Internal and external contacts
- Communication procedures
- Evidence preservation requirements

### Required Contacts

- IT support
- Incident Response Team
- Management
- Business Owners
- Legal Counsel
- Law Enforcement (when appropriate)
- Cyber Insurance Provider

## Example: Ransomware Response

### Step 1 – Isolate

- Disconnect network cables.
- Disable Wi-Fi.
- Remove affected systems from the network.

### Step 2 – Notify

Notify:

- IT Service Provider
- Incident Response Team
- Cyber Insurance Provider

### Step 3 – Preserve Evidence

- Do not power off affected systems unless instructed.
- Preserve logs and forensic evidence.
- Follow regulatory notification requirements.

### Step 4 – Communicate

- Inform employees.
- Notify stakeholders.
- Use approved communication channels.
- Provide clear instructions.

## Practice Your Response

Conduct regular exercises such as:

- Tabletop exercises
- Phishing simulations
- Ransomware scenarios
- Disaster recovery drills

Ensure:

- Every employee knows where the Incident Response Plan is located.
- Employees understand their responsibilities.
- Non-technical staff know how to report suspicious activity.

Review and improve the plan annually.

### Implementation Checklist

- [ ] Develop an Incident Response Plan.
- [ ] Assign response roles.
- [ ] Maintain emergency contacts.
- [ ] Document communication procedures.
- [ ] Preserve forensic evidence procedures.
- [ ] Conduct tabletop exercises.
- [ ] Simulate phishing attacks.
- [ ] Simulate ransomware attacks.
- [ ] Test the response plan annually.
- [ ] Update the plan based on lessons learned.

---

# 6. Recover

Restore operations safely following a cybersecurity incident and improve organizational resilience.

## Objectives

- Validate recovery capabilities.
- Restore business operations.
- Communicate effectively.
- Meet compliance obligations.
- Improve resilience.

### Implementation Checklist

- [ ] Test backup restoration regularly.
- [ ] Maintain Disaster Recovery Plan (DRP).
- [ ] Maintain Business Continuity Plan (BCP).
- [ ] Identify backup infrastructure.
- [ ] Maintain emergency contact lists.
- [ ] Communicate with employees.
- [ ] Communicate with customers when necessary.
- [ ] Notify regulators when required.
- [ ] Meet contractual reporting obligations.
- [ ] Conduct post-incident reviews.
- [ ] Document lessons learned.
- [ ] Improve security controls.
- [ ] Update policies.
- [ ] Update training.
- [ ] Improve incident response procedures.
